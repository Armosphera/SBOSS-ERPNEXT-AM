## Goal
Create the public GitHub repository at `armosphera/SBOSS-ERPNEXT-AM` with the correct description and homepage URL.

## Files to create/modify
- (none — GitHub-side only)

## Contracts to respect
- License: `apps/` will be Armosphera Proprietary; `libs/` will be MIT.
- Description must include the localization scope so search engines and the Frappe partner directory see it correctly.

## Steps

### Step 1 — Create the repo
```bash
gh repo create armosphera/SBOSS-ERPNEXT-AM --public \
  --description "Armosphera ERPNext localization platform — frappe_armenia, frappe_uae, frappe_ai_local apps on top of ERPNext v15.x. Bilingual (HY/AR/EN), local LLM (Gemma 2 2B), Armenia + UAE tax/payroll/e-invoice compliance." \
  --homepage "https://github.com/armosphera/SBOSS-ERPNEXT-AM"
```
Expected: `https://github.com/Armosphera/SBOSS-ERPNEXT-AM`

### Step 2 — Clone locally with `gh` credential helper
```bash
gh auth setup-git   # one-time
cd ~/Desktop
git -c credential.helper= -c "credential.helper=!gh auth git-credential" \
  clone https://github.com/Armosphera/SBOSS-ERPNEXT-AM.git
cd SBOSS-ERPNEXT-AM
```

### Step 3 — Set local git identity
```bash
git config user.email "samvel.stepanyan@armosphera.com"
git config user.name "Samvel Stepanyan"
```

## Verification checklist
- [ ] `gh repo view armosphera/SBOSS-ERPNEXT-AM` shows `Public`, correct description, correct homepage
- [ ] Local clone exists and is empty
- [ ] `git status` clean

## Pitfalls
- **SSH not configured for gh**: use `gh auth setup-git` then the `gh auth git-credential` helper, not literal SSH URLs.
- **Org vs user**: the user is `Armosphera` (capital A); the URL is `Armosphera/SBOSS-ERPNEXT-AM`. The CLI may also accept `armosphera/` (case-insensitive for GitHub).
