```python
# apps/frappe_uae/frappe_uae/regional/doctype/ae_company_setup/ae_company_setup.py
import frappe
from frappe.model.document import Document

JURISDICTIONS = (
    "Mainland", "DIFC", "ADGM", "JAFZA", "DMCC",
    "RAKEZ", "SAIF Zone", "KIZAD", "DAFZA",
    "Abu Dhabi Airport Free Zone", "Other Free Zone",
)

class AECompanySetup(Document):
    def validate(self):
        if not validate_trn(self.tax_registration_number):
            frappe.throw(
                frappe._("UAE TRN must be exactly 15 digits"),
                title=frappe._("Invalid TRN")
            )
        if self.jurisdiction not in JURISDICTIONS:
            frappe.throw(frappe._("Invalid jurisdiction: {0}").format(self.jurisdiction))
        existing = frappe.db.exists("AE Company Setup", {"company": self.company, "name": ["!=", self.name]})
        if existing:
            frappe.throw(
                frappe._("AE Company Setup already exists for {0}").format(self.company),
                frappe.DuplicateEntryError
            )

def validate_trn(trn: str) -> bool:
    if not trn or not isinstance(trn, str):
        return False
    return len(trn) == 15 and trn.isdigit()
```

## Pitfalls
- **Translation files** (W2-T66 covers the full set; add the field labels here too):
  ```csv
  // apps/frappe_uae/frappe_uae/translations/ar.csv
  "AE Company Setup","إعداد شركة الإمارات"
  "TRN (15 digits)","الرقم الضريبي (15 رقم)"
  "Trade License №","رقم الرخصة التجارية"
  ```
- **DIFC/ADGM data laws**: those free zones have separate data-protection laws (excluded from the federal PDPL). This field lets us route them correctly in W2's PDPL-compliance work. Don't validate away free zones.
- **Quarterly vs Monthly VAT**: most UAE SMEs default to quarterly. We default to Quarterly; user can change.
