## Goal
A Docker image at `armosphera/sboss-ollama:gemma4-e2b` that serves Gemma 4 E2B (Q4_K_M) (GGUF, downloaded from Hugging Face) on `localhost:11434`, with health check, model warm-up on start, and a multi-stage build that keeps the final image under 6 GB.

## Contracts to respect
- **Contract B**: the Ollama client only exposes the 4 endpoints in the `AIL Tool Whitelist` (generation, chat, embeddings, list models). Never `delete` or `pull` from agent code.
- **Offline capability**: image must work with no network after the initial `docker pull` of `armosphera/sboss-ollama:gemma4-e2b`. The model is baked in.
- **No telemetry**: Ollama is configured with `OLLAMA_NOPRUNE=1` and `OLLAMA_DEBUG=0` to avoid any analytics callbacks.

### Step 3 — Implement client
```python
# apps/frappe_ai_local/frappe_ai_local/ollama_client.py
from __future__ import annotations
import time
from dataclasses import dataclass
from typing import Iterable
import httpx

@dataclass
class HealthResult:
    ok: bool
    model: str | None
    model_size_bytes: int | None
    error: str | None = None

@dataclass
class ChatMessage:
    role: str   # "system" | "user" | "assistant"
    content: str

@dataclass
class GenerateResult:
    text: str
    latency_ms: int

class OllamaClient:
    def __init__(self, base_url: str, model: str, timeout: float = 60.0):
        self.base_url = base_url.rstrip("/")
        self.model = model
        self._timeout = timeout

    def health(self) -> HealthResult:
        try:
            r = httpx.get(f"{self.base_url}/api/tags", timeout=5.0)
            r.raise_for_status()
            models = r.json().get("models", [])
            m = next((x for x in models if x.get("name") == self.model), None)
            if not m:
                return HealthResult(ok=False, model=None, model_size_bytes=None,
                                    error=f"Model {self.model} not loaded; available: {[x.get('name') for x in models]}")
            return HealthResult(ok=True, model=self.model, model_size_bytes=m.get("size"))
        except Exception as e:
            return HealthResult(ok=False, model=None, model_size_bytes=None, error=str(e))

    def generate(self, prompt: str, max_tokens: int = 512, temperature: float = 0.1) -> GenerateResult:
        body = {"model": self.model, "prompt": prompt, "stream": False,
                "options": {"num_predict": max_tokens, "temperature": temperature}}
        t0 = time.monotonic()
        r = httpx.post(f"{self.base_url}/api/generate", json=body, timeout=self._timeout)
        r.raise_for_status()
        data = r.json()
        return GenerateResult(text=data["response"], latency_ms=int((time.monotonic() - t0) * 1000))

    def _build_chat_body(self, messages: Iterable[ChatMessage]) -> dict:
        return {"model": self.model, "messages": [{"role": m.role, "content": m.content} for m in messages],
                "stream": False}
```

### Step 4 — Build the Docker image
```dockerfile
# infra/docker/ollama.Dockerfile
# Multi-stage: pull model in build stage (slow), ship only runtime + cached model.

ARG OLLAMA_VERSION=0.3.12
FROM ollama/ollama:${OLLAMA_VERSION} AS builder
ARG MODEL=gemma4:e2b
RUN ollama serve & \
    OLLAMA_PID=$! && \
    for i in $(seq 1 30); do ollama list >/dev/null 2>&1 && break; sleep 2; done && \
    ollama pull ${MODEL} && \
    kill $OLLAMA_PID; wait $OLLAMA_PID 2>/dev/null || true

FROM ollama/ollama:${OLLAMA_VERSION}
COPY --from=builder /root/.ollama /root/.ollama
ENV OLLAMA_KEEP_ALIVE=10m \
    OLLAMA_HOST=0.0.0.0:11434 \
    OLLAMA_NOPRUNE=1
EXPOSE 11434
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
    CMD wget -qO- http://localhost:11434/api/tags >/dev/null || exit 1
```

Build & tag:

## Pitfalls
- **Gemma 4 E2B (Q4_K_M) GGUF on Hugging Face**: the file is named `gemma-4-4b-instruct-Q4_K_M.gguf` (verify exact name on HF; the quantization suffix matters — Q4_K_M is the right trade-off for 4B).
- **Ollama model format**: Ollama expects its own manifest format, not raw GGUF. We use `ollama pull` (which handles the conversion) rather than manually copying the GGUF.
- **Build time**: the first build downloads ~2.5 GB of model. Subsequent builds use the cached layer.
- **GPU**: the image works on CPU. For GPU, add `--gpus all` to `docker run`; the multi-stage build is GPU-agnostic.
- **Memory**: 4B model in Q4 needs ~3.5 GB RAM; document 8 GB minimum, 16 GB recommended.
